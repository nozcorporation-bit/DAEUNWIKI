# wiki/ — LLM이 유지하는 합성 영역

`raw/`의 자료를 읽고 추출·정리·연결한 마크다운 페이지가 모이는 곳. **LLM이 이 영역을 전적으로 소유**합니다. 사용자는 읽고, LLM은 씁니다.

## 페이지 카테고리

핵심-맥락.md의 흐름에 맞춘 4개 축 + 인덱스/로그:

### `entities/` — 엔티티 페이지
경쟁사·브랜드·제품·인플루언서. 한 곳에서 그 주체에 대한 모든 것(취급 키워드, 광고 목록, 포지셔닝)을 본다.

### `concepts/` — 개념 페이지
훅(hook) 유형, 후킹포인트 패턴, 설득구조, 전환 논리. **핵심-맥락.md 3번 단계의 분석 결과가 누적되는 곳.** 광고를 분석할 때마다 여기 패턴이 강화됨.

### `keywords/` — 키워드 페이지
키워드별 페이지. 최근 3개월 조회수 추이, 최근 한 달 성장률, 관련 광고, 경쟁 강도. **핵심-맥락.md 1번 단계의 결과가 누적되는 곳.**

### `ads/` — 광고 분석 페이지
개별 광고 한 건당 한 페이지. 영상 링크, 게시일, 조회수, 좋아요, 훅 분석, 후킹포인트, 설득구조, 전환 논리, **우리 제품에 적용 가능한 도출안**. **핵심-맥락.md 2~3번 단계의 결과.**

### `index.md` (필수)
모든 위키 페이지 카탈로그. 카테고리별로 정리, 각 페이지 한 줄 요약. 매 ingest 후 갱신.

### `log.md` (필수)
시간순 append-only 작업 기록.
형식: `## [YYYY-MM-DD] ingest|query|lint|output | 제목`
이렇게 하면 `grep "^## \[" log.md | tail -5`로 최근 활동 확인 가능.

## 페이지 작성 규칙

### Frontmatter (모든 페이지)
```yaml
---
type: entity | concept | keyword | ad
created: 2026-05-09
updated: 2026-05-09
sources: [raw/ads/2026-05-09_브랜드_광고.md, ...]
related: [[다른페이지]]
---
```

### 본문
- 모든 사실 주장에 출처 링크 (`raw/...` 또는 `[[다른위키페이지]]`)
- 페이지 간 상호 링크는 Obsidian 스타일 `[[페이지명]]`
- 모순이 발견되면 `## 쟁점/모순` 섹션 만들어 명시 (지우지 말 것)
- 광고 페이지는 핵심-맥락.md 3번의 항목을 모두 포함:
  - 훅 / 후킹포인트 / 설득구조 / 전환 논리 / **우리 제품 적용 도출안**

## 페이지 생성 트리거

- 어떤 엔티티/개념이 **3회 이상** 언급되면 자체 페이지 생성 검토
- 새 키워드가 `raw/keywords/`에 등장 → `keywords/` 페이지 생성
- 새 광고가 ingest됨 → `ads/` 페이지 생성 + 관련 `entities/`, `keywords/`, `concepts/` 페이지 갱신
- 단일 광고 ingest가 보통 **5~15개 페이지를 touch**합니다. 빠짐없이 갱신하세요.

---

## 유지보수 (Lint)

사용자가 lint를 요청하면 점검:
- 페이지 간 모순
- 새 자료에 의해 superseded된 stale claim
- 인바운드 링크 없는 고아 페이지
- 자주 언급되지만 자체 페이지가 없는 개념
- 광고 페이지에 "적용 도출안"이 빠져 있는 경우 (핵심-맥락.md 3번 미완)
- 키워드 페이지에 최근 30일 데이터가 없는 경우 (stale)
