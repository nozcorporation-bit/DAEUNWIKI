# wiki/ — LLM이 유지하는 합성 영역

`raw/`의 자료를 읽고 추출·정리·연결한 마크다운 페이지가 모이는 곳. **LLM이 이 영역을 전적으로 소유**합니다. 사용자는 읽고, LLM은 씁니다.

## 페이지 카테고리

### `entities/` — 엔티티 페이지
게시자(인플루언서·크리에이터·브랜드·경쟁사). 한 곳에서 그 사람·브랜드의 모든 콘텐츠 목록·포지셔닝·반복 패턴을 본다.

### `refs/` — 레퍼런스 분석 페이지 (메인)
개별 콘텐츠 한 건당 한 페이지. 영상 링크, 게시자, 게시일, engagement, **훅·후킹포인트·설득구조·전환 논리·코르셋 브랜드 적용 도출안 5개 항목 필수**. 핵심-맥락.md 1번 단계의 결과가 누적되는 곳.

### `concepts/` — 개념 페이지
훅 유형, 후킹포인트 패턴, 설득구조, 전환 논리. 같은 패턴이 여러 ref에서 반복 발견되면 자체 페이지로 승격. 도출안의 추출 근거가 누적되는 곳.

### `keywords/` — 키워드 페이지 (런칭 후 우선순위↑)
키워드별 페이지. 최근 3개월 조회수 추이, 최근 한 달 성장률, 관련 콘텐츠, 경쟁 강도. 핵심-맥락.md 2번 단계의 결과.

### `ads/` — 광고 라이브러리 직접 광고 (서브)
광고 라이브러리에서 의도적으로 찾은 광고. 작성 양식은 `refs/`와 동일. 운영의 의도성을 보존하기 위해 refs와 분리.

### `index.md` (필수)
모든 위키 페이지 카탈로그. 카테고리별로 정리, 각 페이지 한 줄 요약. 매 ingest 후 갱신.

### `log.md` (필수)
시간순 append-only 작업 기록.
형식: `## [YYYY-MM-DD] ingest|query|lint|output | 제목`
이렇게 하면 `grep "^## \[" log.md | tail -5`로 최근 활동 확인 가능.

## 페이지 작성 규칙

### Frontmatter (모든 페이지)
```yaml
---
type: entity | ref | concept | keyword | ad
created: 2026-05-09
updated: 2026-05-09
sources: [raw/refs/instagram/2026-05-09_게시자_제목.md, ...]
related: [[다른페이지]]
---
```

### 본문
- 모든 사실 주장에 출처 링크 (`raw/...` 또는 `[[다른위키페이지]]`)
- 페이지 간 상호 링크는 Obsidian 스타일 `[[페이지명]]`
- 모순이 발견되면 `## 쟁점/모순` 섹션 만들어 명시 (지우지 말 것)
- ref/ad 페이지는 5개 항목 빠짐없이:
  - 훅 / 후킹포인트 / 설득구조 / 전환 논리 / **코르셋 브랜드 적용 도출안**

## 페이지 생성 트리거

- 어떤 엔티티/개념이 **3회 이상** 언급되면 자체 페이지 생성 검토
- 새 키워드가 `raw/keywords/`에 등장 → `keywords/` 페이지 생성
- 새 ref/ad가 ingest됨 → `refs/` 또는 `ads/` 페이지 생성 + 관련 `entities/`, `concepts/` 페이지 갱신
- 단일 ingest가 보통 **3~10개 페이지를 touch** (entity + ref + 관련 concept + index + log)

---

## 유지보수 (Lint)

사용자가 lint를 요청하면 점검:
- 페이지 간 모순
- 새 자료에 의해 superseded된 stale claim
- 인바운드 링크 없는 고아 페이지
- 자주 언급되지만 자체 페이지가 없는 개념
- ref/ad 페이지에 "적용 도출안"이 빠져 있는 경우 (핵심-맥락.md 핵심 결과물 미완)
- 키워드 페이지에 최근 30일 데이터가 없는 경우 (stale)
