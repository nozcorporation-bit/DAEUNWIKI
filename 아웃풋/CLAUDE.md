# 아웃풋/ — 최종 산출물 (deliverable)

`wiki/`의 합성을 가공해 만든 결과물. 사용자가 외부에 공유하거나 의사결정·실행에 사용합니다.

## 무엇이 들어가나

핵심-맥락.md의 3단계 흐름 각각이 deliverable로 이어집니다:

- **트렌드 키워드 리포트** — 핵심-맥락.md 1번의 결과물 (월간/주간 급상승 키워드 데이터 표)
- **광고 정리 리포트** — 핵심-맥락.md 2번의 결과물 (광고 리스트 + 게시일/조회수/좋아요 표 + 영상 링크)
- **광고 분석·적용 도출안** — 핵심-맥락.md 3번의 결과물 (훅·후킹·설득·전환 분석 + 우리 제품 적용안)
- **슬라이드 덱** (Marp 형식) — 위 셋 중 무엇이든 발표용으로
- **비교 테이블·차트**

---

## 작성 규칙

### 파일명
- 산출일을 앞에: `2026-05-09_월간-급상승-키워드-리포트.md`
- 분석 대상 명시: `2026-05-09_브랜드명_광고분석.md`

### Frontmatter (필수)
```yaml
---
type: report | analysis | proposal | slides
created: 2026-05-09
scope: "2026-04 월간 / 화장품 카테고리"   # 기간·대상
wiki_sources: [wiki/keywords/...md, wiki/ads/...md]
raw_sources: [raw/keywords/...csv, raw/ads/...md]
---
```

### 인용 규칙
- **모든 산출물은 `wiki/`와 `raw/`를 모두 인용해야 합니다.**
- 합성된 인사이트는 wiki 페이지로, 원본 수치·링크는 raw로 추적 가능해야 함
- 독자가 "이 주장 어디서 왔어?"를 5초 안에 찾을 수 있게

---

## 산출물 ↔ 위키 양방향

- 산출물 작성 중 새 인사이트가 도출되면 **`wiki/`에 역으로 페이지를 추가**하거나 기존 페이지 업데이트
- 같은 분석을 두 번 하지 않도록, 새 산출물 시작 전 기존 산출물·wiki 페이지 확인
- 산출물 완성 후 `wiki/log.md`에 한 줄: `## [YYYY-MM-DD] output | 산출물명`

## 권장 하위 구조 (필요할 때 LLM이 생성)

- `아웃풋/리포트/` — 정기·비정기 리포트 (트렌드 키워드, 광고 정리)
- `아웃풋/분석/` — 광고 분석·적용 도출안
- `아웃풋/슬라이드/` — 발표 자료 (Marp)
- `아웃풋/표·차트/` — 단독 테이블·이미지

## 품질 체크리스트

산출물을 사용자에게 전달하기 전 LLM이 self-check:
- [ ] frontmatter의 `wiki_sources`, `raw_sources`가 모두 채워졌나
- [ ] 핵심-맥락.md의 해당 단계 요구사항을 모두 충족했나
  - 1번: 3개월 vs 1개월 비교, 성장률 수치 명시
  - 2번: 광고별 게시일·조회수·좋아요·링크 정리
  - 3번: 훅 / 후킹포인트 / 설득구조 / 전환 논리 / **우리 제품 적용 도출안** 5개 항목 빠짐없이
- [ ] 모든 수치에 출처 링크
- [ ] 모순되는 데이터가 있다면 별도로 표시 (숨기지 말 것)
