# 다은의 LLM-wiki — 스키마

이 저장소는 Karpathy의 LLM Wiki 패턴을 따릅니다. 사용자는 자료를 큐레이션하고 질문하며, LLM은 위키를 작성·유지보수합니다.

## 핵심 맥락

@핵심-맥락.md

위 파일이 이 위키의 목적과 작업 흐름에 대한 **단일 진실 공급원(single source of truth)** 입니다. `핵심-맥락.md`를 수정하면 모든 세션에서 자동 반영됩니다. 위키 운영 방향이 바뀌면 이 파일을 먼저 고치세요.

---

## 3계층 구조

### 1) `raw/` — 원본 자료 (immutable)
사용자가 큐레이션한 원본. 키워드 데이터 export, 광고 영상 링크/스크린샷/트랜스크립트, 기사 클리핑 등. **LLM은 절대 수정하지 않습니다.** 신뢰의 기반.

### 2) `wiki/` — LLM이 유지하는 합성 영역
원본에서 추출·정리한 마크다운. 엔티티(경쟁사/브랜드/제품), 개념(훅·후킹포인트·전환구조), 키워드 페이지, 광고 분석 페이지. **LLM이 이 영역을 전적으로 소유**하며, 새 자료가 들어올 때마다 관련 페이지를 업데이트하고 상호 참조를 유지합니다.

### 3) `아웃풋/` — 최종 산출물 (deliverable)
외부 공유·의사결정에 쓸 결과물. 트렌드 키워드 리포트, 광고 분석 리포트, 적용 도출안, 슬라이드. wiki/의 합성을 가공해 만듭니다.

---

## 작업 모드

### Ingest (수집)
사용자가 `raw/`에 새 자료를 떨어뜨리면:
1. 자료를 읽고 핵심 takeaway를 사용자와 짧게 논의
2. `wiki/`의 관련 엔티티·개념·키워드·광고 페이지를 업데이트
3. 새 페이지가 필요하면 생성하고 `wiki/index.md`에 등재
4. 핵심-맥락.md의 3단계 흐름(키워드 추출 → 광고 정리 → 훅·전환 분석)에 어디에 해당하는지 분류
5. `wiki/log.md`에 한 줄 기록: `## [YYYY-MM-DD] ingest | 자료명`

### Query (질의)
1. `wiki/index.md`를 먼저 읽고 관련 페이지 식별
2. 해당 페이지들을 읽고 답변 합성, 출처 링크 포함
3. 답변이 재사용 가치가 있으면 `wiki/` 또는 `아웃풋/`에 보존 제안

### Lint (점검)
주기적으로 사용자가 요청하면:
- 페이지 간 모순
- 새 자료에 의해 갱신되어야 할 stale claim
- 인바운드 링크 없는 고아 페이지
- 3회 이상 언급되지만 자체 페이지가 없는 개념
- 빠진 상호 참조, 데이터 갭(웹 검색으로 보강 가능한)

### Output (산출)
사용자가 deliverable을 요청하면 `wiki/` 페이지를 종합해 `아웃풋/`에 작성. 산출물은 `wiki/`와 `raw/`를 모두 인용.

---

## 인덱스와 로그

- `wiki/index.md` — 모든 위키 페이지 카탈로그 (카테고리별, 한 줄 요약). 매 ingest 후 갱신.
- `wiki/log.md` — 시간순 append-only 기록. 형식: `## [YYYY-MM-DD] ingest|query|lint|output | 제목`

두 파일은 첫 ingest 시 LLM이 생성합니다.

---

## 컨벤션

- 위키 페이지는 한국어로 작성 (사용자가 영어 요청 시 영어)
- 파일명: 한국어 허용, 공백은 하이픈(`-`)으로
- 페이지 상호 링크: Obsidian 스타일 `[[페이지명]]`
- 날짜: 절대 표기 (`2026-05-09`), "어제/지난주" 같은 상대 표기 금지
- 사실 주장에는 출처 링크 필수 (`raw/...` 경로 또는 다른 wiki 페이지)
- 위키 페이지 상단 YAML frontmatter:
  ```yaml
  ---
  type: entity | concept | keyword | ad | report
  created: 2026-05-09
  updated: 2026-05-09
  sources: [raw/.../파일.md]
  ---
  ```

---

## 폴더별 세부 규칙
각 하위 폴더에는 자체 `CLAUDE.md`가 있습니다. 그 폴더에서 작업할 때는 해당 파일의 규칙도 함께 따르세요.
- `raw/CLAUDE.md` — 원본 자료 보존 규칙
- `wiki/CLAUDE.md` — 합성 페이지 작성·유지 규칙
- `아웃풋/CLAUDE.md` — 산출물 작성·인용 규칙
